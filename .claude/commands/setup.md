# Setup - Validate Environment

Validate that the environment is properly configured for the workflow.

## Overview

This command checks:
1. Git availability
2. Required MCP servers
3. File structure
4. Security issues
5. Environment configuration

## Instructions

This command is typically auto-generated by `/quick-start` as part of the workflow project.

When user runs this command, check:

### 1. Git Status
```bash
git --version
```

If git not available:
- Explain why git is needed (version control, collaboration, history)
- Offer to help install
- Provide installation instructions for their OS

### 2. MCP Servers

Check which MCPs are configured in `.claude/settings.local.json`:
- Read the `enabledMcpjsonServers` list
- For each required MCP (from project design):
  - Check if enabled
  - Try a simple test call
  - Report status

**Example**:
```
Checking MCP servers...
✅ notion - Connected
✅ slack - Connected
❌ jira - Not configured
   Install: npm install -g @atlassian/mcp-jira
   Configure in .claude/settings.local.json
```

### 3. File Structure

Verify expected directories exist:
- `.claude/commands/`
- `.claude/skills/` or `.claude/agents/`
- `.claude/knowledge/`
- `project-plan/`
- Session directories

### 4. Security Check

Invoke `security-checker` skill to scan for:
- Hardcoded secrets
- Missing .gitignore patterns
- Sensitive files in git

### 5. Environment Variables

If project uses .env:
- Check .env.example exists
- Check .env exists (don't read contents)
- Verify required variables documented

### Report

Show comprehensive status:
```
🔧 Environment Setup Status

Git: ✅ version 2.39.0
MCPs: ⚠️  2/3 configured (jira missing)
Files: ✅ All directories present
Security: ✅ No issues found
Environment: ✅ .env configured

Action needed:
- Install Jira MCP (see above)

Once complete, run your workflow: /[workflow-name]
```

## User Experience

This is a diagnostic command that helps troubleshoot setup issues. Be helpful and specific about how to fix problems.
