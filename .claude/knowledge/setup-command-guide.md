# Setup Command Implementation Guide

This guide explains how to implement the `/setup` command that gets generated for user workflows.

## Purpose

The `/setup` command is created by `/quick-start` in each user's workflow project. It helps new users (or the original user in a new environment) validate their environment is ready to run the workflow.

## Overview

When a user runs `/setup` in their workflow, check:
1. Git availability
2. Required MCP servers (specific to their workflow)
3. File structure
4. Security issues
5. Environment configuration
6. Local permissions configuration

## Implementation Instructions

When a user runs the generated `/setup` command in their workflow, Claude should:

### 1. Git Status
```bash
git --version
```

If git not available:
- Explain why git is needed (version control, collaboration, history)
- Offer to help install
- Provide installation instructions for their OS

### 2. MCP Servers

**This section is auto-generated by `/quick-start`** based on the workflow's data source requirements.

The workflow-specific `/setup` command (in the user's project) will list required MCPs. Use `ListMcpResourcesTool` to check availability:

1. Call `ListMcpResourcesTool()` without parameters to list all available servers
2. For each required MCP listed in the workflow's setup command:
   - Check if server name appears in the results
   - Verify it's responding (resources returned = working)
   - Report status with setup instructions

**IMPORTANT**: If MCPs are missing, provide clear instructions but don't block:
- Show installation commands (e.g., `npm install -g @modelcontextprotocol/server-notion`)
- Explain configuration locations (global `~/.claude/settings.json` or local `.claude/settings.local.json`)
- Document required credentials (API keys, OAuth setup)
- **Note**: Workflow may support manual data input if MCPs unavailable (check workflow documentation)

This works with both global MCPs (in ~/.claude/settings.json) and local MCPs (in .claude/settings.local.json).

**Example**:
```
Checking MCP servers...
✅ notion - Connected (12 resources available)
✅ slack - Connected (8 resources available)
❌ jira - Not found
   Install: npm install -g @atlassian/mcp-jira
   Configure in ~/.claude/settings.json or .claude/settings.local.json
```

### 3. File Structure

Verify expected directories exist:
- `.claude/commands/`
- `.claude/skills/` or `.claude/agents/`
- `.claude/knowledge/`
- `project-plan/`
- Session directories

### 4. Security Check

Invoke `security-checker` skill to scan for:
- Hardcoded secrets
- Missing .gitignore patterns
- Sensitive files in git

### 5. Environment Variables

If project uses .env:
- Check .env.example exists
- Check .env exists (don't read contents)
- Verify required variables documented

### Report

Show comprehensive status:
```
🔧 Environment Setup Status

Git: ✅ version 2.39.0
MCPs: ⚠️  2/3 configured (jira missing)
Files: ✅ All directories present
Security: ✅ No issues found
Environment: ✅ .env configured

Action needed:
- Install Jira MCP (see above)

Once complete, run your workflow: /[workflow-name]
```

## User Experience

This is a diagnostic command that helps troubleshoot setup issues. Be helpful and specific about how to fix problems.
